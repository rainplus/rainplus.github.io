java8 3/18/2014

接口默认方法与静态方法

/**
 * 默认方法
 */
public interface DefMethod {
    /**
     * 已知一个接口的的属性都会被声明为 public final static
     * 而方法只会被声明为 public abstract
     *
     *  java8 现在给接口提供了默认实现,使用 default 关键字进行声明
     *  当实现这个接口的类型未进行覆盖方法里，会默认使用这个实现
     *
     *  如果是旧的方法，实现类必须实现这个接口的全部方法否着只能声明为 abstract 类
     */

    default void method(){
        // do something
    }
    void abstractMethod();
}

lambda 表达式

lambda 是希腊的第十一个字母（大写Λ，小写λ），高中知识：物理上的波长符号。 在 java 中的应用是简化内部类的写法。但 lambda 表达式并不能表达所有的匿名类，只能用户取代函数接口

举栗子：

  1. 无参函数的简写

旧方式：

new Thread(new Runnable(){
    @Override
    public void run(){
      // do something
    } 
});

使用 lambda 表达式：

new Thread(()->{
    //do something
});
  1. 带参数的简写

如果要给一个字符串列表自定义比较器，按照字符串长度进行排序，在 java8 之前的写法：

List<String> list = Arrays.asList("one","two","three","four");
Collections.sort(list,new Comparator<String>(){
    @Override
    public int compare(String s1,String s2){
        if(s1 == null)
            return -1;
        if(s2 == null)
            return 1;
        return s1.length()-s2.length();
    }
})

使用 java8 进行编写：

List<String> list = Arrays.asList("one","two","three","four");
Collections.sort(list,(s1,s2)->{
        if(s1 == null)
            return -1;
        if(s2 == null)
            return 1;
         return s1.length()-s2.length();});

函数接口（Functional Interface）

  1. 函数接口定义指一个内部类中只有一个抽象的接口，强类型推断;（可以声明 default 接口）
  2. 新增加注解：@FunctionInterface 显示化声明，编译检查。（建议显示声明，只是给人看）

一般只要满足条件条件1 就认为是一个函数接口

可以操作一个对象中的公开的方法与构造器，通过限制符(::)进行引用

Object o = new Object();
// 旧的方法

lambda 变量作用域

传递给 lambda 的变量，因为 lambda 本身与匿名类有一定的相似性，所以变量的传递是也是基本一致的。

  • 接收本地变量 
    final int num =1;
Converter(Integer,String) stringConverter = (from) ->String.valueOf(from->num);
stringConverter.convert(2); 
// 但如果不进行声明 final 也是可以的，代码也是有效的，
// 前提条件是 num 在这个作用域下是隐式地 final 
// 如果后面修改到 num 这个变量的值，这段代码就不会编译通过
  • 接收静态变量与类属性 lambda 可以对类属性与静态变量进行读写
  • 接收接口默认方法

内置的函数式接口

java8 提供了大量的函数式接口，通过对旧的类（Comparator、Runnable）实现@FunctionalInterface 注解进行支持函数式变化 ， 同时也提供了大量的新的函数式接口。

  • Predicates（断言） 断言类函数式是返回 boolean 类型判断的函数。提供了各种复杂的逻辑判断。

    Predicate<String> predicate = (s)->s.length() > 0;
// 转变回 java8 之前的形式：
      Predicate<String> predicate = new Predicate<String>() {
           @Override
           public boolean test(String s) {
               return s.length() > 0;
           }
       };
 // 断言类的内容 还提供了很多实现
 predicate.test("foo");              // true
 predicate.negate().test("foo");     // false
   
 Predicate<Boolean> nonNull = Objects::nonNull;
 Predicate<Boolean> isNull = Objects::isNull;
   
 Predicate<String> isEmpty = String::isEmpty;
 Predicate<String> isNotEmpty = isEmpty.negate();

  • Functions 功能性的函数式接口：接受一个参数并生成一个结果，默认方法提供了一个链式的功能。

Function<String, Integer> toInteger = Integer::valueOf;
Function<String, String> backToString = toInteger.andThen(String::valueOf);

backToString.apply("123");     // "123"
  • Suppliers（提供器） 不传递参数，但会生成一个结果，构造器就是一种 Suppliers
Supplier<Person> personSupplier = Person::new;
personSupplier.get();   // new Person
  • Consumers（消费者类） 只传递一个参数 ，但没有结果返回 ，只作中间过程处理
Consumer<Person> greeter = (p) -> System.out.println("Hello, " + p.firstName);
greeter.accept(new Person("Luke", "Skywalker"));
  • Comparators

Optionals （比较器）

Comparator<Person> comparator = (p1, p2) -> p1.firstName.compareTo(p2.firstName);

Person p1 = new Person("John", "Doe");
Person p2 = new Person("Alice", "Wonderland");

comparator.compare(p1, p2);             // > 0
comparator.reversed().compare(p1, p2);  // < 0

Optionals

Optional 并不是一个函数式接口，只是一个漂亮的防止 NullPointerException 的方式。

Base64 集成

package com.javacodegeeks.java8.base64;

import java.nio.charset.StandardCharsets;
import java.util.Base64;

public class Base64s {
    public static void main(String[] args) {
        final String text = "Base64 finally in Java 8!";
        
        final String encoded = Base64
            .getEncoder()
            .encodeToString( text.getBytes( StandardCharsets.UTF_8 ) );
        System.out.println( encoded );
        
        final String decoded = new String( 
            Base64.getDecoder().decode( encoded ),
            StandardCharsets.UTF_8 );
        System.out.println( decoded );
    }
}
